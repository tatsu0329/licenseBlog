# ページ構成戦略：需要と収益性の分析

## コンテンツ需要マトリクス

### 高需要 × 高収益（最優先）
1. **過去問解説ページ**
   - 需要：★★★★★（検索ボリューム大、直前期の流入多い）
   - 収益：★★★★★（アプリDL導線が最も効果的）
   - 戦略：CTAを複数配置、アプリ内へのディープリンク

2. **勉強ロードマップ**
   - 需要：★★★★（学習開始時の検索）
   - 収益：★★★★（診断ツール→アプリ、PDF導線）
   - 戦略：診断ツール誘導→アプリDL

### 高需要 × 中収益
3. **試験概要・難易度・合格率**
   - 需要：★★★★★（初心者入口、検索ボリューム最大）
   - 収益：★★★（CTA弱め、アプリ導線）
   - 戦略：軽いアプリCTA、ロードマップへの導線

4. **FAQ**
   - 需要：★★★★（検索意図明確）
   - 収益：★★（間接的な導線のみ）
   - 戦略：内部リンク強化、アプリ紹介への導線

### 中需要 × 高収益
5. **アプリ紹介ページ**
   - 需要：★★★（直接検索は少ない）
   - 収益：★★★★★（CV直接）
   - 戦略：過去問・ロードマップからの導線強化

6. **診断ツール**
   - 需要：★★★（学習中のニーズ）
   - 収益：★★★★（アプリプレミアム導線）
   - 戦略：無料体験→有料保存・詳細分析

### 中需要 × 中収益
7. **出題傾向分析**
   - 需要：★★★（中上級者の検索）
   - 収益：★★★（プレミアム詳細分析への導線）
   - 戦略：データ差別化、有料詳細への導線

8. **よくある間違い**
   - 需要：★★★（SNS拡散で流入）
   - 収益：★★（間接的）
   - 戦略：SNS最適化、アプリ導線

### 低需要 × 低収益（資産型）
9. **試験制度変更**
   - 需要：★★（更新時のみ）
   - 収益：★（間接的）
   - 戦略：長期SEO資産として維持

10. **Web vs アプリ**
    - 需要：★★（新規流入少ない）
    - 収益：★★（合理性訴求）
    - 戦略：アプリ紹介ページへの内部リンク

## ページ構成の優先順位とCTA配置

### 1級整備士トップページ（最重要）
**構成順序（上から下）：**
1. H1 + メタ説明（SEO）
2. 試験概要クイックビュー（高需要、CTA弱め）
3. 合格率推移グラフ（視覚的価値）
4. **アプリCTA（中）** ← ここに最初のCTA
5. メインコンテンツリンク（需要順）
   - 過去問（高需要×高収益）→ 強調
   - 勉強ロードマップ（高需要×高収益）→ 強調
   - 出題傾向（中需要×中収益）
6. その他コンテンツ（中需要）
7. **アプリCTA（強）** ← 最後にもCTA

### 過去問解説ページ（最高収益）
**構成順序：**
1. 問題文
2. **アプリCTA（強）** ← 「この問題をアプリで解く」
3. 選択肢
4. 正解・解説
5. **アプリCTA（中）** ← 「関連問題をアプリで解く」
6. 関連問題リンク
7. **アプリCTA（弱）** ← フッター付近

### 勉強ロードマップページ
**構成順序：**
1. 全体像（価値提供）
2. 分野別重要度
3. **診断ツールCTA** ← 学習計画診断
4. 過去問ベース逆算学習法
5. 学習優先度判断
6. 社会人/学生別プラン
7. **アプリCTA（中）** ← 学習管理用アプリ
8. PDF導線（有料化の余地）

### 試験概要ページ
**構成順序：**
1. 資格の概要
2. 受験資格
3. 難易度・合格率（視覚化）
4. 合格率推移
5. 勉強時間の目安
6. **軽いアプリCTA** ← 「学習計画を立てる」
7. ロードマップ・過去問への導線

## CTA配置戦略

### CTA強度の定義
- **強**：大型、目立つ色、複数回配置
- **中**：標準サイズ、適度な配置
- **弱**：控えめ、フッター付近

### 各ページのCTA配置
1. **過去問解説**：強×2、中×1、弱×1 = 計4回
2. **勉強ロードマップ**：中×2、診断ツール×1 = 計3回
3. **アプリ紹介**：強×3 = 計3回（CVページ）
4. **試験概要**：弱×1 = 計1回
5. **出題傾向**：中×1（プレミアム詳細）= 計1回
6. **診断ツール**：中×1（アプリ詳細診断）= 計1回

## 収益化フロントローディング戦略

### 優先表示するコンテンツ
1. **高需要×高収益**：過去問、ロードマップ
2. **高需要×中収益**：試験概要、FAQ
3. **中需要×高収益**：アプリ紹介、診断ツール

### 内部リンク戦略
- 過去問ページ → アプリ紹介（強）
- ロードマップ → 診断ツール → アプリ（強）
- 試験概要 → ロードマップ → アプリ（中）
- FAQ → ロードマップ / 過去問 → アプリ（弱）

## マネタイズ導線の最適化

### 無料 → 有料の導線
1. **過去問閲覧** → アプリDL（無料10問/日） → プレミアム
2. **診断ツール使用** → 詳細診断・保存（有料） → アプリ
3. **出題傾向閲覧** → 詳細分析PDF（有料） → アプリ
4. **ロードマップ閲覧** → PDFダウンロード（有料） → アプリ

### 合理性訴求のポイント
- 「Webで調べる、アプリで解く」の役割分担を明確化
- 無料でできることと有料の価値を明確に区別
- 強引な誘導ではなく、必要な時に使える選択肢を提示
